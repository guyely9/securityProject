# securityProject

פרויקט מחקר - קורס 20940 באוניברסיטה הפתוחה - מבוא לאבטחת המרחב המקוון

סקירת הפרוייקט:
פרוייקט זה בוחן את העמידות של מנגונני אימות שונים בפני מתקפת ניחוש סיסמאות (brute force, password spraying). במהלך הפרוייקט השווינו בין אלגוריתי גיבוב שונים ובחנו את ההשפעה של שכבות הגנה אקטיביות על אבטחת המערכת. 

בהתאם לדרישות המטלה, הבטחנו את מקוריות המחקר באמצעות מפתח קבוצתי ייחודי המורכב מxor בין תעודות הזהות שלנו:
group_seed = 527740437
המפתח מוטמע בקובץ הנתונים של המשתמשים, בתיקית התוצאות בשורות הלוגים וביצירת סיסמה ייחודית עבור אחד המשתמשים בניסוי. 

הסיסמאות של המשתמשים הוגדרו כך: 10 סיסמאות קלות- סיסמה קלה היא סיסמה נפוצה שמכילה סדר מספרים נפוץ מילים נפוצות, משפטים מוכרים, ללא שילוב בין אותיות למספרים
10 סיסמאות בינוניות - סיסמה בינונית היא סיסמה שמכילה שילוב בין מספרים לאותיות אבל בתחום ההיגיון, כלומר לא אקראי אלא עם משמעות כלשהי, בנוסף אחת הסיסמאות מכילה את GROUP SEED
10 סיסמאות קשות- סיסמה קשה מכילה שילוב אקראי של מספרים, אותיות, סימני מקלדת , ללא משמעות קריאה, רנדומלית

המערכת מדמה שרת אימות המממש את המנגוננים הבאים:
1. אלגוריתמי גיבוב: SHA-256, bcyrpt ו- argon2id
2. הגנות:
  נעילת חשבון לאחר 5 נסיונות כושלים - account lockout
  הגבלת מספר הבקשות ליחידת זמן - Rate limiting
  אימות דו שלבי מבוסס זמן - TOTP
  מנגנון הגנה בפני בוטים -captcha

הוראות הרצה:
1. דרישות קדם:

1.1 ודא שמותקנת אצלך סביבת python 3.11 ומעלה

1.2 יש להתקין את הפסריות הנדרשות באמצעות הפקודה:

```pip install bcrypt argon2-cffi pyotp pillo```

1.3 הפרוייקט מתבסס על קובץ rockyou.txt המכיל סיסמאות נפוצות.
ניתן להוריד את קובץ הסיסמאות בקישור: 
https://weakpass.com/wordlists/rockyou.txt

ודא שהקובץ rockyou.txt נמצא בתיקיית השורש של הפרויקט

2. בחירת ההגדרות הרצויות:
בקובץ config.py מגדירים את אלגוריתם הגיבוב ומדליקים/מכבים את ההגנות האקטיביות עבור כל הרצה. 
להוסיף פה את הקוד


3. הרצת השרת:
השרת מדמה את מערכת האימות עם ההגנות השונות. לכל הרצה יש להשתמש בקובץ ה-entry point:

```python app.py```

4. יצירת בסיס הנתונים, המשתמשים וקובץ הסיסמאות לתוקף
הקובץ  setup_experiment.py אחראי על יצירת המשאבים לתקיפה ולהגנה
הריצו בטרמינל:

```python setup_experiment.py```

5. הרצת המתקפות
המתקפות מבוצעות באופן אוטמוטי על ידי השרת כחלק מתהליך ניסוי בעזרת brute force ו passwords sprayin.
הריצו את הקובץ attacker:

```python attacker.py```
 
*אם תרצו לתקוף רק בעזרת brute force או רק passwords spaying, ניתן להפוך את השורה בmain שקוראת לסוג התקיפה שאינכם מעוניינים בה להערה. 

6. צפייה בתוצאות
בסיום כל הרצה, יווצר קובץ פלט של לוגים בתיקייה חדשה (אלא אם כן כבר קיימת) בשם logs
שם הקובץ נקבע לפי שיטת הגיבוב וההגנות הפועלות בו. 
כדי להפוך את הלוגים לטבלה מסודרת וקריאה יותר, הריצו את הקובץ logs_to_csv:

```python logs_to_csv.py```

הקובץ ייפתח תקייה חדשה (אלא אם כן כבר קיימת) בשם results, שם יווצר קובץ csv הרצוי. 

אם תרצו להשוות בין הניסויים באופן ויזואלי, מהיר וממצת. הריצו את הקובץ analyze_results:

```python analyze_results.py```

קובץ זה יכין 4 גרפים וישמור אותם בתיקייה results.

- השוואה בין מהירויות של תקיפות על גיבובים שונים
- כמות ההצלחות עבור כל הגנה
- אחוזי ההצלחות עבור כל הגנה
